# 스마트스토어 주문 → Todoist 2개 태스크

스마트스토어 출고 엑셀 한 개를 받아 Todoist에 **두 개의 부모 태스크**를 만든다. 발송 작업할 때 "뭘 몇 개 사야 하나(합산)"와 "누구한테 뭘 보내나(수취인별)"를 따로 보고 싶어서 둘로 나눈다.

- **태스크 1 — `사야할 제품들`**: 모든 주문을 상품(상품번호+옵션) 기준으로 합산. 서브태스크 = `상품명 × 합산수량`. 사입/장보기용.
- **태스크 2 — `수취인별 주문`**: 수취인 = 서브태스크, 그 사람 품목 = 그 아래 서브-서브태스크. 포장/발송용.

상품명은 **한글(엑셀의 상품명 컬럼) 그대로** 쓴다. 영문 변환·학습 사전 없음 — 사용자가 한글로 충분하다고 확인했다.

## 사용법

인자: 자유 형식. 다음을 추출하고 부족한 것만 사용자에게 묻는다.

- **파일경로** (스마트스토어 출고 엑셀 .xlsx) — 없으면 최근 업로드 파일을 쓴다.
- **비밀번호** (스마트스토어 파일은 보통 비번 걸려 있음. 없으면 빈 문자열. 학습된 기본값: 1111)
- **마감일** (예: `오늘`, `내일`, `2026-06-02`) — 두 부모 태스크에 동일하게 적용.
- **프로젝트명** (기본값: `스마트스토어`)

## 실행 단계

### 1. 엑셀 파싱 (스크립트)

```bash
pip3 install msoffcrypto-tool pandas openpyxl xlrd -q
python3 <스킬폴더>/scripts/parse.py "<엑셀경로>" "<비밀번호>"
```

표준출력은 JSON 한 줄:

```json
{
  "shopping": [ {"product_id": "...", "option": "맛: Chunk", "kor_name": "...", "qty": 1} ],
  "recipients": [ {"name": "한성주", "items": [ {"product_id":"...","option":"","kor_name":"...","qty":1} ]} ],
  "buyer_count": 5,
  "shopping_line_count": 9
}
```

- `shopping` = 상품(상품번호+옵션) 합산. 같은 상품이라도 옵션이 다르면 별도 라인.
- `recipients` = 수취인명 기준 묶음(원본 행 순서 보존). 같은 (수취인, 상품, 옵션, 주문번호)는 수량 합산.

### 2. 표시 문구 만들기

각 품목의 서브태스크 텍스트는 다음 형식으로 만든다:

- 옵션 없음: `{kor_name} × {qty}`
- 옵션 있음: `{kor_name} ({옵션값}) × {qty}` — 옵션값은 `맛: Chunk`에서 `: ` 뒤만 (`Chunk`).

`kor_name`이 길면(예: `캐나다 코스트코 네슬레 킷캣 F1 레이싱카 밀크초콜릿 693g`) 사용자가 알아볼 수 있게 앞쪽 원산지/유통사 수식어를 가볍게 줄여도 된다(예: `킷캣 F1 레이싱카 밀크초콜릿 693g`). 핵심 식별 정보(브랜드·제품·용량·옵션)는 남긴다. 한글은 유지. 확신이 안 서면 원문 그대로 둔다.

### 3. 미리보기 + 사용자 확인

JSON을 사람이 읽기 쉽게 풀어 보여준다:

- **태스크 1 사야할 제품들**: 합산 품목 리스트 (라인 수 표시)
- **태스크 2 수취인별 주문**: 수취인별로 들여쓰기, 각자 품목 (수취인 수 표시)

마감일·프로젝트명도 같이 표시. **"그대로 만들어줘" / "GO" 류 응답 전엔 Todoist 호출 X.**
마감일·프로젝트가 안 정해졌으면 이때 묻는다.

### 4. Todoist 프로젝트 확정 + (추가 주문이면) 기존 대조

`find-projects`로 프로젝트명 검색. 없으면 사용자에게 "새로 만들까요?" 묻고 `add-projects`로 생성.

> **추가 주문(추가분/"추가된거만") 처리 시 — 중복 방지 필수 (2026-07-06 학습):**
> 기존 배치에 주문을 덧붙일 때는 반드시 **완료(crossed off / checked)된 태스크까지 대조**한다.
> `find-tasks`는 미완료만 반환하므로, `find-completed-tasks`(parentId 지정, since/until 넉넉히)도
> 같이 조회해서 기존 수취인·품목을 확인할 것. 사용자는 발송하면서 항목을 하나씩 완료 처리하므로,
> 완료된 것을 못 보고 다시 넣으면 중복이 난다. **진짜 신규 = 미완료에도 완료에도 없는 것만** 추가한다.
> 추가 주문이면 부모(사야할 제품들/수취인별 주문)를 새로 만들지 말고, 확인한 기존 부모 아래에 신규분만 붙인다.

### 5. 부모 태스크 2개 생성 (신규 배치일 때만)

`add-tasks`로 `사야할 제품들`, `수취인별 주문` 두 부모를 한 번에 생성. `dueString`은 사용자가 준 마감일을 **ISO(`YYYY-MM-DD`)로 환산**해 넣는다 — Todoist 자연어 파서가 한국어 토큰(`내일` 등)을 못 알아듣는 경우가 있다. 반환된 두 부모의 `id`를 기억한다.

### 6. 서브태스크 생성

- **태스크 1**: `shopping`의 각 품목을 `사야할 제품들` 부모(`parentId`) 아래 서브태스크로.
- **태스크 2**: 먼저 `recipients`의 수취인명들을 `수취인별 주문` 부모 아래 서브태스크로 만들고, 반환된 각 수취인 `id`를 받아 그 아래에 품목들을 서브-서브태스크(`parentId` = 수취인 id)로 넣는다.

> **타임아웃 주의:** Todoist MCP는 한 호출에 태스크가 많으면 서버 타임아웃이 잘 난다. **한 호출당 4~5개**로 나눠라. 호출이 실패(`server isn't responding`)하면 같은 묶음을 그대로 한 번 더 보내면 대개 성공한다. 수취인 서브태스크를 먼저 다 만든 뒤(그래야 id를 받음) 품목을 채운다.

### 7. 결과 보고

생성한 부모 2개(또는 추가한 신규분), 태스크1 서브 수, 수취인 수 + 그 아래 품목 수, 프로젝트명을 간단히 알리고 끝. Todoist 앱에서 직접 검토·수정해 달라고 안내.

## 참고

- 수취인은 `수취인명` 기준. 비어 있으면 `구매자명`으로 폴백. (선물 주문이면 구매자≠수취인일 수 있음 — 발송 라벨 기준인 수취인을 쓴다.)
- Todoist MCP가 연결돼 있지 않으면 먼저 connector 카드를 띄워 연결을 부탁한 뒤 진행.
- 같은 입력 엑셀을 쓰는 자매 스킬: 영문명·구매자별 단일 태스크는 [[order-list-todoist]], 워드 문서는 [[order-list]].
