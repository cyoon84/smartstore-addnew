캐나다(온타리오) 상품 매입 시 세금 처리 규칙. **자동으로 판단하지 않고 매번 사용자에게 확인**한다. 사용자도 잘 모르는 경우가 있어 안전하게 확인하는 게 원칙.

## 핵심 원칙

1. **사용자가 제공하는 가격은 세전(pre-tax)이다.** 표시가/매장가 그대로 받음. 세금은 별도 계산.
2. **세금은 매번 묻는다.** 카테고리만 보고 자동 적용 금지. 5가지 필수 확인 항목 중 "세금" 단계에서 AskUserQuestion으로 확인.
3. 사용자가 명시적으로 "이 카테고리는 항상 X%로 가" 같이 자동화를 요청한 경우만 예외.

## 참고 세율 (사용자 확인용)

물어볼 때 후보로 제시할 수 있는 일반적인 패턴 (확정 사실 아님 — 사용자 판단이 우선):

| 카테고리 | 보통 세율 |
| --- | --- |
| 일반 상품 (가공식품·캔디·스낵·음료·생활용품 등) | **HST 13%** |
| 소스 (마요네즈·머스타드·케첩·드레싱 등) | **0%** |
| 커피류 (캡슐·원두·분쇄) | **0%** |
| 쿠키 | **0%** |
| 베이킹/팬케이크·와플 믹스 (밀가루·기초 식료품류) | **0%** (2026-05-24 Flourish 단백질 팬케이크 믹스 케이스 — 사용자 정정으로 확인) |
| 키즈 의류 (kids size) | **GST 5%** |
| 신발 | **GST 5%** |

> 키즈 의류·신발은 HST 0%가 아니라 **PST 면제 + GST 5%만 적용**이라 5%.

## 가격 계산식

세전 가격 기준으로 일관:

```
cost_pre_tax = 사용자 제공 가격 (CAD)
cost_with_tax = cost_pre_tax × (1 + tax_rate)        (실매입가)
sell_pre_fee = cost_with_tax + markup                (마진 적용)
sell_original = sell_pre_fee / (1 - 0.066)           (네이버 수수료 gross-up)
sell_krw = ceil_10(sell_original × exchange_rate)
```

세율(`tax_rate`)별:
- 0% → `cost_with_tax = cost_pre_tax` (× 1.00)
- 5% → × 1.05
- 13% → × 1.13

`product_info.json`의 `pricing` 블록에 `cost_pre_tax`, `tax_rate`, `cost_with_tax` 함께 기록.

## 워크플로 통합

§4 워크플로 §1 "사용자 제공 입력 확인" 단계에서 세금 질문은 항상 포함:

> "세금은 몇 % 적용? (참고: 일반 13% / 소스·커피·쿠키 0% / 키즈 의류·신발 5%)"

사용자가 "안 붙어 / 0%" → tax_applied: false. "13% / 5%" → tax_applied: true, tax_rate에 기록.
