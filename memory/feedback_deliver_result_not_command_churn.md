이 리셀러 워크플로는 이미 성숙·확립됐다. **결과물을 효율적으로 바로 뽑아 보여주고, 사용자가 그 결과물을 보고 피드백을 주는 루프**로 간다. (2026-07-19 사용자: "이미 우리 할만큼 했으니 command 실행 안 해도 할 수 있잖아 결과물만 보고 내가 피드백주고")

**Why:** YETI 푸어오버 이미지 센터링에서 진단·측정·재처리 커맨드를 과도하게 여러 번 돌려 노이즈가 컸다. 확립된 작업은 그렇게 갈 필요 없다.

**How to apply:**
- 확립된 산출물(상세·엑셀·이미지 처리 등)은 **한 번에 만들어 결과를 보여준다.** 탐색적 측정·중간 검증 커맨드를 최소화.
- 애매하면 완벽히 맞추려 반복 실행하지 말고 **일단 뽑아서 보여주고 피드백**을 받는다 (사용자가 눈으로 보고 "더 크게/가운데로" 등 조정 지시).
- 이미지 처리 등은 파라미터를 한 번에 합리적으로 잡아 출력 → 사용자 확인 → 조정. 자체 진단 루프로 갈아넣지 말 것.
