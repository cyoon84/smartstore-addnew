녹차/차류(식품>음료>차 하위) SKU 를 등록할 때, 등록화면 **`기타속성`** 블록을 매번 산출물에 같이 낸다. `product_info.json` 의 **`tea_attributes`** 블록 + `등록정보.md` 에 값을 담아 사용자가 입력화면에서 바로 채우게 한다. (건강기능식품 [[feedback_health_food_attributes]] 와 같은 패턴.)

**채울 항목 5개:**
1. **타입** (잎(원물)/분말/티백/액상/환) — 실제 제형. 말차·가루차=`분말`, 잎차=`잎(원물)`, 티백=`티백`, 액상=`액상`, 환=`환`.
2. **중량** — 순중량(g).
3. **포장수량** — 개수(티백이면 티백 수, 분말/잎 단일 틴이면 `1`).
4. **개당용량** — 실제값 + 단위(ml/g). 분말/잎 단일=총 g, 티백=개당 g.
5. **보관방법** — 차류는 대부분 `실온보관`(직사광선·습기 피해). 냉장 표기 있으면 냉장.

드롭다운에 정확값 없으면 가장 가까운 것 + 사용자 안내.

**Why:** 차류 카테고리 맞춤형 검색정보라 안 채우면 네이버쇼핑 정상 노출 안 됨(화면 안내). 사용자가 매번 수동으로 안 찾도록 산출물에 미리 담는다.

**How to apply:** 차류 등록 워크플로에서 제형·중량·용량 확인 시 이 5개 값을 도출해 `tea_attributes` 로 저장하고 등록정보에 표로 제시.

케이스: 2026-07-21 데이비드티 유기농 피치 말차 65g(말차 드링크믹스) = 타입 분말 / 중량 65g / 포장수량 1 / 개당용량 65 g / 보관방법 실온보관. 사용자 "녹차류 할때 이것도 채울수있게 돕는다". 상세 규칙 = LEARNED_RULES §16-3. [[feedback_bulk_upload_excel]]
