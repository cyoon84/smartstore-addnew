# Todoist 추가 주문 — 완료 항목까지 대조해 중복 방지 (+ 부모 새로 만들지 말 것, 2026-07-11 재발)

스마트스토어 출고 주문을 Todoist에 정리하는 워크플로(order-2task-todoist / order-list-todoist 스킬)에서,
**추가 주문("추가분", "추가된거만")을 기존 배치에 덧붙일 때** 반드시 지킬 것.

**🔑 2026-07-11 재발 — 원인은 "검색을 안 함":** 7/9에 이미 만들어둔 무제목 `사야할 제품들` 부모(due 7/17)가
있는데도, 화요일(7/14) 배치를 처리하며 기존 부모를 검색하지 않고 `사야할 제품들 (7/14 출고)`라는 **날짜 붙은
새 부모**를 만들었다. 날짜를 붙이면 "별개 배치처럼" 보여서 기존 부모와 다른 것으로 착각하기 쉽다 — 그러나
아래 규칙은 "부모를 새로 만들지 말 것"이지 "날짜 붙이면 예외"가 아니다. 사용자가 "니가 사야할 제품들
(7/14 출고) 새로 또 만드는 바람에 이 사달이 난거잖아" · "그 전에도 이거때문에 난리나서 메모리에 저장했잖아!"로
지적 — 학습이 스킬 실행 절차에 실제로 안 붙어 있었던 게 근본 원인. **Why:** 이렇게 지적하는 이유는, 동일
주문(예: 김하나의 네스카페)이 어느 부모 소속인지 사용자와 에이전트 둘 다 헷갈려 대화가 여러 턴 낭비됐기 때문.

**🔑 사용자 명시 지시(2026-07-11) — "화/금에 부모 태스크가 이미 생성되어있으면 그 밑에다 계속 넣어":**
배치·출고일 단위로 부모를 나눌지 말지 **판단하지 말 것**. 화요일/금요일 어느 출고일이든, 검색해서
`사야할 제품들`/`수취인별 주문` 부모가 하나라도 있으면 **무조건 그 밑에 이어서 넣는다.** 새 부모는
프로젝트에 그 이름의 부모가 **정말 하나도 없을 때만** 만든다.

**적용:** `add-tasks`로 부모 생성하기 **전에 무조건** `find-tasks(projectId, searchText:"사야할 제품들")` /
`find-tasks(projectId, searchText:"수취인별 주문")` 로 기존 부모 검색 → 있으면 그 `id` 재사용(이름에 날짜
접미사 붙이지 않음), 검색 결과가 0건일 때만 스킬 기본형 이름으로 새로 생성.

## 핵심 규칙 (2026-07-06 중복 실수에서 학습)
- 기존 태스크 대조 시 **완료(crossed off / checked)된 태스크까지 포함**해서 확인한다.
- `find-tasks`는 기본적으로 **미완료만 반환** → 이미 처리(완료)한 수취인·품목이 안 보인다.
- 사용자는 발송하면서 수취인/품목을 하나씩 crossed off(완료) 처리하므로, 완료된 걸 못 보고 다시 넣으면 **중복**이 난다.
- 그래서 대조할 때 `find-completed-tasks`(parentId 지정, since/until 넉넉히)도 **같이** 조회한다.
- **진짜 신규 = 미완료에도 완료에도 없는 수취인/품목만.** 그것만 추가한다.
- 추가 주문이면 부모("사야할 제품들"/"수취인별 주문")를 새로 만들지 말고, 확인한 **기존 부모 아래에 신규분만** 붙인다. (새 부모를 또 만들면 중복 배치가 됨.)
