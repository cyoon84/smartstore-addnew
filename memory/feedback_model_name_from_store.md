dispatch(폰 → 데스크탑) 또는 일반 메시지로 가격·마진 정보를 줄 때, 사용자가 **"지금 어느
매장에서 보고 있는지"**(현재 보고 있는 매장)를 함께 적는다. 에이전트는 그 매장을 소스로
**공식 제품명(영어 풀네임)을 추출**해서 네이버 **모델명** 필드를 채운다.

## 흐름

1. dispatch `스토어` 필드 = **현재 보고 있는 매장** (Costco / Walmart / Loblaws / Amazon CA·US 등).
   단순 출처 라벨이 아니라 **공식 제품명을 끌어올 소스 매장**으로 쓴다.
2. 그 매장(URL 또는 매장 사이트 검색)에서 **official full product name** 추출.
3. 추출한 공식 영어 풀네임을 **모델명 필드**에 넣는다 (CLAUDE.md §5 모델명 규칙).
   - 모델명 = 영어 원래 제품명(공식 풀네임). 영어/로마자 검색 매칭 + 적합도↑.
   - 그룹 상품은 **옵션별 모델명 따로**.
4. `product_info.json` 의 `model_name` 필드와 `등록정보.md` 모델명 항목에 기록.

## 데이터 진실성 (§9 직결)

- **크롤 가능 환경**(URL 입력 · Cowork dispatch 등 매장 사이트 접근 가능) → 소스에서 **직접 추출**.
- **크롤 차단 / 사진만 있는 환경**(데스크톱 현장 사진만) → 공식명을 추측해 단정하지 말고
  **사용자에게 소스 확인 요청** 후 채운다. 확실치 않으면 모델명에 "확인要" 표기.
- 매장 표기와 실제 공식 브랜드 풀네임이 다르면 공식 풀네임 우선(매장 약식 표기 X).

## 예

- 입력: `[사진] Costco / $16.99 / 13% / $3 / 개당 5000 / single`
  → Costco 사이트에서 공식 제품명 "Kirkland Signature Ancient Grains Granola 1kg" 추출
  → 모델명 = `Kirkland Signature Ancient Grains Granola 1kg`.
