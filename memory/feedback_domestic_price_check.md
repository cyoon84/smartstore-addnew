스마트스토어 신규 상품 등록 워크플로 시작 전에 **반드시 먼저** 다음을 수행한다:

1. **쿠팡 검색** — 동일/유사 SKU가 국내 정식 유통(특히 로켓배송)으로 팔리는지 확인
2. **네이버 쇼핑 검색** — 동일 SKU의 가격대 / 셀러 수 / 최저가 확인
3. **시세 평가 후 보고:**
   - 한국 동일 SKU 최저가 (G마켓·11번가·쿠팡·롯데홈쇼핑 등 채널별)
   - 내가 책정할 가격(상품가+배송비 실수령 기준)과의 차이
   - "가격 경쟁력 있음 / 박빙 / 가격으로 안 됨" 한 줄 판단
4. 사용자에게 **A) 진행 B) 가격 조정 C) 접고 다른 SKU** 선택지 제시
5. 명시적 GO 받은 뒤에 product_info.json / detail.html / 등록정보.md 산출물 생성 시작

**Why:** 박한 마진($2 등)으로 책정한 가격이 한국 정식 유통(쿠팡 로켓·홈쇼핑 폭탄가) 대비 경쟁력 없으면 등록·발송 노력 전체가 매몰비용. 사용자 명시 원칙 — "가격으로 안 되면 걍 접는다." (2026-05-17 Snuggle 드라이어시트 케이스에서 학습: 책정가 ₩13,900+배송 ₩18,900 vs 쿠팡 로켓 ₩20,000, 롯데홈쇼핑 ₩12,900 무료 — 같은 브랜드 안에 폭탄가 채널 있으면 노출에서 밀림).

**How to apply:**
- 입력 데이터(사진+가격+세금+마진+배송) 받자마자 **가격 산출 직전** 또는 **직후** 검색 수행
- 검색 막혀있는 도구는 사용 가능한 대체 경로(Google 검색 → 캐시된 결과)로 우회
- 시세 미확인 상태로 산출물 4종 만들지 않는다 — 사용자가 접으면 재작업
- "캐나다 직수입 / 코스트코 한정" 같은 차별점이 가격차를 이길 수 있는지도 같이 평가
- 사용자가 명시적으로 "시세 확인 스킵하고 바로 진행"이라고 한 경우는 예외
