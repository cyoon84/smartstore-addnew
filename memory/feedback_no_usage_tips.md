스마트스토어 상세페이지 만들 때 다음 종류의 섹션을 **자동 생성하지 말 것**:
- "활용 팁" / "사용 팁" / "이렇게 드세요"
- "추천 레시피" / "조리 예시"
- 사용 시나리오, 페어링 제안 등 LLM 추측 기반 콘텐츠

**유지해도 되는 섹션** (브랜드/스토어 페이지에서 가져온 검증된 정보):
- 상품 설명 (스토어 페이지에서 추출한 description)
- 상품 포인트 / 특징 (브랜드가 명시한 features, certifications)
- 제품 사양 (브랜드/용량/원산지/보관방법)
- 원재료 / 알레르기 (라벨 또는 브랜드 페이지 출처)
- 배송 안내 (사용자 규칙)

**Why:** 사용자(리셀러)가 상세페이지를 고객용으로 그대로 올림. 검증 안 된 활용 팁은 부정확하거나 오해 소지가 있어 컴플레인/반품 리스크. §9 데이터 진실성 원칙과 직결.

**How to apply:** 모든 product-detail-page-ko 작업에서 기본적으로 활용 팁 섹션 생략. 사용자가 명시적으로 "활용 팁도 넣어줘" / "레시피 아이디어 추가해" 라고 요청한 경우에만 추가.
