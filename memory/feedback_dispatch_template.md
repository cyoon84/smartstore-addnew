사용자가 Claude Dispatch(폰 → 데스크탑) 또는 일반 메시지로 상품 사진과 함께 아래 형식의 입력을 보내면, **추가 질문 없이 곧바로 워크플로 진입**한다.

## 받아들이는 두 가지 형식

### 1. 블록 형식 (라벨:값)
```
[사진 첨부]
스토어: <store_name>
가격: $<amount> (세전)
세금: <0% / 5% / 13%>
마진: <$amount 또는 X%>
배송: <e.g. 2개당 15000>
모드: <single / group / append>
엔진: agents          ← (선택) 있으면 서브에이전트 모델로 실행, 없으면 기본 워크플로
```

### 2. 슬래시 한 줄 형식
```
[사진 첨부]
<스토어> / <가격> / <세금> / <마진> / <배송> / <모드>
```
예: `Loblaws / $6.29 / 0% / 15% / 2개당 15000 / single`
서브에이전트 모델로 실행하려면 끝에 토큰 추가: `... / single / agents` (또는 `에이전트`)

## 파싱 규칙

- **스토어** — **현재 보고 있는 매장** (Costco, Walmart, Loblaws, Amazon CA/US, AliExpress 등. 약자도 인정: WMT, COSTCO). 단순 출처 라벨이 아니라 **공식 제품명을 추출할 소스 매장** — 그 매장에서 영어 풀네임을 뽑아 **모델명** 필드를 채운다 ([[feedback_model_name_from_store]])
- **가격** — 통화 기호 없으면 CAD 가정. "세전"이 디폴트(어차피 사용자 제공 가격은 세전)
- **세금** — `0%`, `5%`, `13%`. 숫자만 와도 % 가정
- **마진** — `$5`(절대값) 또는 `15%`(마크업 비율). 둘 중 하나 명시
- **배송** — `2개당 15000`, `개당 3000`, `포함` 등
- **모드** — single / group / append. 누락 시 single 기본
- **엔진** — (선택) `agents`/`에이전트` 토큰이 있으면 서브에이전트 모델(`.claude/commands/register-agents.md` 오케스트레이션)로 실행. 없으면 기존 기본 워크플로. 누락은 기본 워크플로 가정, 별도로 안 물음

## 누락 처리

- 5가지 중 **누락된 항목만** AskUserQuestion으로 한 번에 물음
- 모드 누락은 single로 가정하고 별도 안 묻기
- 사진에서 상품 식별 실패하면 "어떤 상품?" 한 번 물음

## 동작

위 형식이 감지되면:
1. 사진 분석으로 상품명/브랜드 추출 (§2.5)
2. 지정 스토어(= 현재 보고 있는 매장)에서 검색 → 후보 매칭 → **공식 제품명(영어 풀네임) 추출 → 모델명 필드** (그룹은 옵션별, [[feedback_model_name_from_store]])
3. 페이지 크롤 → 데이터 추출
4. 가격 계산 (이미 메모리에 있는 6.6% 수수료 + 환율 + 반올림 규칙 적용)
5. `/Volumes/External/claude/smartstore-project/output/`에 `<slug>_detail.html`, `<slug>_product_info.json`, `<slug>_등록정보.md` 평탄 저장 (자세한 규칙: [[feedback_output_location]])
6. 결과 파일 링크 제공

각 단계 시작·완료를 TaskList에 기록해 사용자가 진행 상황 볼 수 있게 함.

## 주의

- 사진+텍스트 외에 모호한 표현이 추가로 있으면(예: "예쁘게 만들어줘") 무시
- 사용자가 "확인해줘"라고 명시한 단계가 있으면 그 단계에서 멈추고 보고
