**배송비를 손익에서 "구매자 부담이라 상쇄"로 무시하지 말 것.** 받는 배송비 > 실제 해외배송원가면 차액이 마진이고, **네이버 수수료가 제품가엔 6.6%지만 배송비엔 3.63%만** 붙어 배송비 쪽 실수령이 더 크다.

**수수료 구성 (정산내역 확인):** **6.6% = Npay 결제수수료 3.63% + 매출연동 수수료 3.0%.** 매출연동(3.0%)은 **배송비엔 미적용(0)** → 배송비는 Npay 3.63%만. (정산 예: 제품 ₩29,900 → −1,084(Npay)−897(매출연동)=−1,981 → ₩27,919 / 배송비 ₩15,000 → −543(Npay)−0 = ₩14,457.)

**산식(개당):**
```
배송_실수령 = 배송비_받는것 × (1 − 0.0363)      # 배송비 네이버 수수료 3.63% (제품가는 6.6%)
배송_마진   = 배송_실수령 − 실제_해외배송원가KRW
총_실이익   = (판매가×0.934 − 원가KRW) + 배송_마진
```

**예 (다우니 언스토퍼블 1.13kg, 2026-07-07):** 배송비 ₩15,000 × 0.9637 = ₩14,456 실수령 − 실배송원가($11.25/1.5kg×1083=₩12,184) = **배송마진 +₩2,272/개**. 제품가가 판매가 대비 본전(인스타카트 원가 ₩28,570)이어도 배송마진이 갭 메꿔 총 +₩1,629 로 전환.

**How to apply:** SKU 마진 판단 시 항상 **제품손익(수수료 6.6%) + 배송손익(수수료 3.63%)** 합산. 배송비를 실비+마진으로 넉넉히 받는 구조면 저마진 제품도 총합 플러스 가능. 실배송원가는 무게식([[feedback_overseas_shipping_baseline]]) — 1.5kg 은 선형 $9+$4.5×0.5=$11.25. [[project_inventory_list]]

**🔑 효자 SKU 판별 = 가벼운 무게 + 넉넉한 마크업 (2026-07-07 "이런게 효자제품"):** 배송비는 정액(₩15,000)인데 실배송비는 무게 비례($9/1kg급·$11.25/1.5kg) → **가벼울수록 배송마진↑.** + 원가 대비 판매가 여유(제품 마크업)까지 있으면 양쪽에서 남는 효자. 무겁고 마크업 얇으면 배송마진도 작고 제품도 본전 → 딜/글리치 아니면 별로. **소싱 시 (a)가볍고 (b)원가 대비 판매가 여유 있는 품목 우선.**
- 예: 다우니 퓨전 303g(원가$11·판매₩23,000·실배송$9) = 제품+₩8,013+배송+₩4,710 = **+₩12,722/개(효자)** vs 1.13kg(매장$24.99·판매₩31,900·실배송$11.25) = −₩795+₩2,273 = +₩1,478/개(본전).

**Why:** 2026-07-07 사용자: "다우니 언스토퍼블 1.13kg는 배송비 15000원씩 받아 / 실제 배송료 1.5kg-11.25$ / 배송비에서는 네이버 수수료 3.63%만 적용됨." 배송마진을 빼먹으면 저마진 SKU 를 실제보다 나쁘게 판단. 무게가 배송마진을 좌우하므로 가벼운 고마크업 품목이 효자.
