영양제/건강기능식품 SKU 를 등록할 때, 네이버 등록화면에 뜨는 **`건강식품 인증 필수`** 블록과 **기타속성**을 매번 산출물에 같이 낸다. `product_info.json` 의 **`health_food_attributes`** 블록 + `등록정보.md` 에 값을 담아, 사용자가 입력화면에서 바로 채우게 한다.

**채울 항목 8개:**
1. **주요 기능성** (드롭다운: 영양보충/**관절/뼈건강**/눈건강/면역력/간건강/피로회복 등) — 🔑 **가장 구체적인 기능성 우선.** `관절/뼈건강` 옵션이 있으니 **칼슘·비타민D 등 뼈·치아 제품 = `관절/뼈건강`**(영양보충 아님). 눈=눈건강, 면역=면역력, 간=간건강, 피로=피로회복. **`영양보충`은 특정 기능 없는 일반 미네랄/비타민 보충일 때만**(예: 마그네슘). 드롭다운 다 훑고 세부 기능 있으면 그걸 먼저(영양보충으로 뭉개지 말 것).
2. **영양소 원료명(식약처고시)** — 성분표 기준 든 성분 체크박스 선택. 칼슘+비타민D3 → `칼슘`·`비타민D` 둘 다 / 마그네슘 단일 → `마그네슘`.
3. **섭취대상** — 성인/어린이 등(라벨).
4. **제품타입** — 캡슐/정제/구미/분말(실제 제형).
5. **제품용량** — 🔑 N정 아님, **"며칠분"(일분/개월분) 드롭다운.** 총 정수 ÷ 1일 총 섭취량 = 일수 → 최근접 옵션(30일=1개월). 예 마그네슘 300정÷1=300일=`10개월분` / 칼슘 350정÷2=175일≈`6개월분`.
6. **1일 총 섭취량** — 🔑 **단일 숫자 드롭다운(1·2·3·4…), 범위 선택 불가.** 라벨이 범위(2~4정)면 표준·최소치 단일값(→ `2`).
7. **섭취횟수** — 단일 숫자(1일 몇 회). 칼슘처럼 나눠먹는 건 총 섭취량 `2`+횟수 `2`(1회 1정)로 코히런트하게.
8. **섭취방법** — 보통 `물과 함께 섭취`.

드롭다운에 정확값 없으면 가장 가까운 것 + 사용자 안내(없는 값 억지 삽입 금지).

**Why:** 건강기능식품은 이 속성이 등록 필수라 안 채우면 등록이 막힌다. 사용자가 매번 수동으로 찾지 않도록 산출물에 미리 담는다.

**How to apply:** 영양제/건강기능식품 등록 워크플로에서 성분표(성분·제형·dosage) 확인 시 이 8개 값을 도출해 `health_food_attributes` 로 저장하고 등록정보에 표로 제시.
