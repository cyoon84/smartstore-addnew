기존에 낱개로 등록된 단일상품들을 통합하기로 할 때, 두 방식은 산출물 준비 단계부터 다르다.

- **그룹상품으로 갈 제품군** → 일괄등록 엑셀은 기존과 동일하게 옵션(맛/향/색상)마다 **개별 단일상품 행**으로 준비한다(옵션마다 별도 상품번호로 일괄등록). **그룹으로 묶는 작업(그룹상품 전환) 자체는 자동화하지 않고 사용자가 등록 후 스마트스토어 판매자센터에서 직접 수행**한다.
- **"한 상품 + 여러 옵션"으로 갈 제품군** → 말 그대로 **하나의 상품ID 안에 옵션값 여러 개를 다 담아** 일괄등록 엑셀 한 행으로 준비한다(옵션 컬럼에 맛/색상 여러 값을 다중 옵션으로 기입).

**Why:** **네이버 일괄등록(엑셀 업로드) 자체가 아직 그룹상품 생성을 지원하지 않는다** (2026-07-10 사용자 확인) — 그래서 옵션마다 개별 단일상품으로 일단 일괄등록하고, 그룹으로 묶는 작업은 등록 후 스마트스토어 판매자센터 UI에서 수동으로만 가능하다. build_bulk_excel.py 같은 자동화 도구가 그룹 연결까지 하려고 시도하지 않는 건 단순히 범위 밖이라서가 아니라, **일괄등록 API/엑셀 자체에 그룹 생성 기능이 없기 때문**. 반면 "한 상품+여러옵션"은 일괄등록 엑셀의 옵션 컬럼에 값 여러 개를 채우는 것만으로 등록 시점에 완결되는 작업이라 자동화 범위 안에 있다.

**How to apply:** 단일상품 통합 후보를 추릴 때, "이 제품군은 그룹으로 갈지 / 한 상품+여러옵션으로 갈지"부터 사용자에게 확인하고, 그 결정에 따라 위 두 준비 방식 중 하나로 일괄등록 산출물을 만든다. 그룹 전환 자체를 대신 해주겠다고 하지 않는다.
