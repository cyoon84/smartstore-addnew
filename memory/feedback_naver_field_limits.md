2026-07-05 룰루레몬 클리어보틀 32oz 등록에서 연속으로 겪은 "삐빅"을 종합. 등록 산출물 만들 때 아래를 미리 지켜 재작업을 줄인다.

## 필드 글자수·특수문자 제약

| 필드 | 제약 | 대응 |
|---|---|---|
| **상품명** | 50자 초과 시 "검색 최적화 수정 제안" 태클(하드블록 아님, 100자까지 저장은 됨) | 50자 미만 권장. **그룹/옵션 상품이면 색상을 상품명에서 빼고 옵션값으로** → 그룹명 짧게. 색상별 단일상품이면 **한글 색상명**이 영문보다 짧아 45자 내 가능 |
| **모델명**(상품정보제공고시) | **50자 미만**("50자보다 작게") + **금지문자 `\ * ? " < >`** | `build_bulk_excel.clean_gosi_model()` 이 색상 접미·브랜드 접두 제거 + 금지문자(별표 등) 제거 + 49자 컷. 룰루 `*Straw Lid` 의 `*` 가 걸림 |
| **옵션값** | **25자 미만**("옵션값N 25자보다 작게") | 긴 색상조합은 24자 이내로. **한글 음역이 영문보다 짧음**(영문 최대 24 vs 한글 최대 14). 조합형 옵션값은 콤마(,) 구분이라 값 안에 `/` 는 OK |
| **옵션명** | 최대 25자, 여러개는 줄바꿈(엔터) 구분 | 색상 단일축이면 `색상` |

## 이미지 URL
- **셀러 관리자 도메인 `sell.smartstore.naver.com/...` URL 은 상세 img 로 차단**("허용되지 않는 URL"). → **Flickr(`live.staticflickr.com`) 등 외부 호스팅**으로 재업로드. lululemon CDN(`images.lululemon.com`)·네이버쇼핑 노출 CDN(`shopping-phinf`)은 대체로 허용. [[feedback_bulk_upload_excel]] · [[feedback_detail_html_bare_fragment]]

## 상세 HTML 태그
- 커머스 API 필터: **전역 영향 태그만 차단 — `HTML·META·SCRIPT·STYLE(블록)·BODY·HEAD`.** `div·span·inline style=·h1·h2·img·p·strong·br` 은 허용(에디터·API 둘 다). [[feedback_detail_styled_deco_template]]
- **`word-break:keep-all`** 안쪽 카드 div 에 넣어 단어 중간 끊김 방지(전 하위 상속). 데스크탑/모바일 렌더 테스트로 확인.

## 등록 구조 선택 — 단일 / 옵션(조합형) / 그룹
- **색상별 따로따로 단일상품**: 각 색상 독립 리스팅(리뷰·노출 각각). 상품명에 색상 포함 → **한글 색상명이면 45자 내라 태클 회피**. 색상별 상세 각 1장(자기 색상 + 공식 풀네임 병기). 일괄엑셀 = 색상당 1행.
- **단일상품 + 옵션(조합형)**: 한 상품ID·한 리스팅에 9색 옵션. 상품명=색상 없는 그룹명(짧음), 옵션값=색상(25자 제한). **상세 1장에 전 색상** 넣어야 함(옵션명 pill + 공식 풀네임). `build_bulk_excel` 이 product_info `options[]` → 1행 조합형(옵션값/가/재고 콤마).
- **그룹상품**: 여러 상품 묶기(§13-1). 그룹명+옵션 합침 노출에 **반복 단어 다수면 거부** → 옵션값을 겹치지 않는 짧은 고유명으로.
- 셀러가 상황따라 선택(같은 SKU도 전환 가능). 사용자 지시 우선.

> 룰: 등록 산출물 만들 때 상품명 50·모델명 50/특수문자·옵션값 25·이미지 외부호스팅·상세 전역태그금지 5개를 사전 점검. 어긴 채 만들면 사용자가 업로드에서 삐빅 → 재작업.

## 🔑 옵션명 = "속성: 값" 형식 (2026 AI 알고리즘, 2026-07-11 스터디)
2026년 AI 검색 개편 이후 **옵션명이 구조화되어 있으면(예: `색상: 화이트`, `사이즈: M`, `수량: 1개입`) 노출 가중치가 붙고, 어수선한 옵션명(`1번`·`A`·`이거 색깔`)은 무시된다**는 최적화 가이드 확인(tokki-ai 블로그, 자사솔루션 홍보성이라 수치는 걸러내되 방향은 채택). 우리 옵션값(색상 한글명·용량 등)은 이미 값 자체는 구조적이지만, **옵션명 라벨을 "색상"·"용량"·"수량" 처럼 속성명으로 명확히 달 것** — `build_bulk_excel.py`에 넘기는 product_info `options[]`의 옵션명 필드를 점검. (신뢰도: 방향 타당, 수치·알고리즘 디테일은 근거 약함.)

## 참고 — 신호 우선순위 (2026-07-11 스터디, i-boss/tokki-ai 종합)
카테고리(1위, 오분류면 나머지 다 고쳐도 노출 안 됨) → 상품명(2위) → 옵션(3위, 구조화 가중치) → 태그(4위, 노이즈 제거 위주) → 상세페이지(5위, 가장 오래 걸리지만 매출 직결). 이미 §15 카테고리·§5 상품명·§18 필드제약 순으로 엄수해온 것과 방향 일치 — 새 절차 변경 없음, 우선순위 재확인 차원.

## 🔑 Flickr 이미지 네이버 수집 순간 실패 → 재시도 (2026-07-07)
일괄등록 업로드 시 대표/추가이미지(Flickr `live.staticflickr.com` URL)가 **"[추가이미지] 유효하지 않거나 존재하지 않는 이미지입니다"** 로 뜰 수 있다. 원인은 URL·해상도 문제가 아니라(둘 다 정상이어도 발생) **네이버 이미지 수집기의 순간 실패**(핫링크/레이트리밋 추정) — 같은 배치의 대표이미지는 통과하는데 추가이미지 1장만 걸리는 식. **삭제하지 말고 잠시 후 재업로드하면 통과**한다(2026-07-07 스타벅스 GLS 콜드컵 케이스: 처음 실패 → 재시도 성공). 추가이미지는 필수 아님이라 계속 막히면 빼도 되지만, 먼저 재시도.
