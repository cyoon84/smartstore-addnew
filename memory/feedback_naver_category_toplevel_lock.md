# 네이버 대(최상위)분류 카테고리는 등록 후 수정 불가 → 복사 후 새 상품 재등록

네이버 스마트스토어는 **이미 등록된 상품의 대(최상위) 카테고리를 다른 대분류로 변경할 수 없다.**

- **같은 대분류 내 이동** (예: `패션잡화 > A > B` → `패션잡화 > C > D`) → 기존 listing에서 즉시 수정 가능.
- **대분류 자체가 달라지는 이동** (예: `스포츠/레저 …` → `패션잡화 …`) → **기존 listing에서 변경 불가.**

## 해결

대분류가 바뀌어야 하면: 기존 제품을 **복사 → 올바른 카테고리 적용 → 새로운 상품으로 등록**한다. 기존 잘못된 listing은 판매중지/정리. (셀러 수동 작업)

## SEO 리프레시 시사점

seo-auditor가 카테고리 오분류를 잡고 listing-writer가 새 카테고리를 제안할 때, 그 이동이 **대분류(최상위)를 넘나드는지** 반드시 표시한다:

- 같은 대분류 내 → "카테고리 수정(즉시 가능)"
- **대분류 변경 → "⚠️ 기존 상품 수정 불가 — 복사 후 새 상품 등록 필요(셀러 작업)"** 를 Slack 요약·등록정보.md에 명시.

## 사례

2026-06-02 나이키 짐 토트백 SEO 리프레시 — 기존 `스포츠/레저 > 축구 > 축구가방`(오분류)을 `패션잡화 > 여성가방 > 토트백`으로 바꾸려니 대분류(스포츠/레저 → 패션잡화)가 달라 수정 불가. 셀러가 기존 제품 복사 후 새 카테고리로 재등록. (listing-writer 제안은 보스턴백이었으나 실제 적용은 토트백.)
