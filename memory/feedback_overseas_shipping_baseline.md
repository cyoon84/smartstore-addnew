캐나다→한국 **해외 배송비는 무게 기준**. 기본이 **$9부터** = 편의상 **₩10,000** baseline.

**산정식:** `총무게 = 단위중량 × 수량 → kg 올림(ceil)`, `배송원가($) = 9 + 4.5×(올림kg−1)`.
- 1kg → $9 ≈ **₩10,000** / 2kg → $13.5 ≈ **₩15,000** / 3kg → $18 ≈ **₩20,000** (편의상 환산·라운드)
- 네이버 **"수량별 부과수량(M개당)"** = 무게가 1kg 경계를 넘기는 개수. 예) 765g 제품 → 2개=1.53kg→2kg=₩15,000 → **2개당 ₩15,000**(개당 7,500).

**규칙:**
- **사용자가 배송비를 따로 언급하지 않으면 → 개당 ₩10,000(1kg baseline) 디폴트.** ₩10,000 밑으로는 안 잡음(하한).
- 사용자가 명시한 배송비(개당 N원 · M개당 N원 · 무배+흡수 등)가 있으면 그게 **최우선**. 무게식·baseline 은 미지정·하한 가드.

**Why:** 실제 캐나다→한국 해외 배송 원가가 첫 1kg $9 + 추가 1kg당 $4.5 구조라, 그 밑으로 받으면 배송에서 손해. 이전 `/source-launch` 임시 기본 ₩15,000(§14)은 저단가·부피 작은 품목의 도착가 경쟁력을 죽였고, 실제 디폴트·하한은 ₩10,000($9)·그 위는 무게식.

**How to apply:** dispatch·일반 등록·source-launch 어디서든 배송비 입력이 없거나 애매하면 제품 무게로 위 식 적용(미지정·1kg 이하면 개당 ₩10,000). build_bulk_excel 의 product_info `shipping` 에 `"M개당 N원"`(2개당 15000 등)·`"개당 N원"`·`{per_unit_krw:N}` 형태로 기입하면 수량별 fee·부과수량 자동. 명시값 있으면 그대로.

**🔑 커피머그(도자기 머그류) 특례 = $1 통관수수료 없음 + `$8 + $2.25/500g` (2026-07-20):** 실배송비 계산 시 **커피/도자기 머그류는 $1 통관수수료가 안 붙어** base 가 $9 아닌 **$8**, 증가분은 **500g당 $2.25**(= 표준 $4.5/kg 동일). 즉 표준식에서 base −$1 만 다름. 예) 이석주 스타벅스 미니머그 1kg → **$8**(₩8,664) — 가볍고 통관수수료 없어 배송 흑자(₩10,000 받아 +₩973). 정산 실배송비에 반영(부피무게 아닌 실무게·머그류 특례 우선). 일반 품목은 여전히 표준 $9 base. 어느 게 통관수수료 붙고 안 붙는지 애매하면 사용자 확인.
