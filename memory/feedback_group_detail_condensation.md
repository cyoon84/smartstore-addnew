그룹상품 detail.html에서 **옵션 안내**, **사이즈 안내**는 본문에 길게 풀어 쓰지 말고 **시각 카드 캡션 수준의 한 줄**로 응축한다. 사용자가 네이버 에디터에서 이미지 카드(사이드-바이-사이드 박스 사진, callout 박스 등) 안에 넣어 시각적으로 강조하기 때문.

**Why:** 2026-05-26 Quaker Harvest Crunch 그룹상품 케이스에서 학습. 제가 쓴 detail.html은:
- "맛 옵션" 섹션: 오리지널 1단락 설명 + "추후 다른 맛 추가 예정" 1단락
- "용량 안내" 섹션: 475g/1.8kg 각각 1단락 + 권장 사용자 시나리오

사용자가 네이버 등록 시 모두 응축:
- 맛 옵션: "오리지널(Original) — 가장 클래식한 본연의 맛, 두가지 용량중 골라보세요" 한 줄
- 용량 안내: 475g·1.8kg 박스 사진 2장 사이드-바이-사이드 + "475g"/"1.8kg 대용량" 짧은 캡션만

**How to apply:**
- 그룹상품 detail.html에서 옵션·사이즈 설명은 **2~3줄 이내**로 짧게 작성
- "추후 다른 옵션 추가 예정" 같은 미래형 안내는 1줄로 압축하거나 생략 (네이버 본문에 미래 약속 적기 부담)
- 시각 카드에 들어갈 가능성 큰 정보(사이즈·용량·옵션값)는 **캡션 한 줄** 수준으로 쓰고, 길게 풀고 싶은 정보는 "이런 점이 좋아요" / "이런 분들께 추천해요" 섹션에 분산
- 헤드라인은 한글 + 영문 부제 2줄로 [[feedback_detail_english_subtitle]] 룰 그대로 유지 (사용자가 callout 박스로 시각화)
- 본문 강조용 USP(FIFA 월드컵 26 한정 등)는 이미지에서 드러나면 본문 강조 줄여도 OK
