네이버 스마트스토어 운영을 매일 조금씩 개선하기 위해, **최신 정보를 검색·학습하고 우리 워크플로와 대조**해
Slack 으로 보고한다. 사용자 지시(2026-07-06): "매일 저녁마다 구글/네이버에서 스마트스토어 관련 검색 후
공부하면서 개선해보자." 입력(선택): $ARGUMENTS · 배경: [[project_daily_smartstore_study]]

## 파이프라인

1. **중복 방지 로드** — `output/cron_logs/study_log.csv`(없으면 헤더 `date,topic,source_url,takeaway` 생성)를
   읽어 **이미 학습한 URL/주제**를 파악한다. 오늘은 안 다룬 주제를 고른다.

2. **주제 선정 (로테이션)** — 인자로 주제가 오면 그걸, 없으면 아래 축에서 오늘 요일 기준으로 하나:
   상세페이지/체류시간 · 상품명 SEO · 태그/키워드 · 카테고리 · 네이버 AI검색 알고리즘 ·
   리뷰/셀러답변 · 등록정책(검토필요·필수필드) · 구매대행/해외직구 규정 · 시즌 트렌드.

3. **검색** — WebSearch + (막히면) insane-search 스킬 + 네이버 검색 MCP(`search_blog`·`search_news`·
   `search_cafearticle`)로 **최근 글** 검색. 톡키AI 블로그(tokki-ai.com/blog) 신규글도 주기적으로 확인.
   신뢰도: 자사솔루션 홍보글이어도 방향 타당하면 채택하되 과장 걸러냄.

4. **정독** — 유용한 글 **2~3편**을 WebFetch 로 정독해 실행 가능한 포인트 추출(수치·예시 인용).

5. **우리 현황과 대조** — 추출 포인트를 `docs/LEARNED_RULES.md`(§5 상품명·§7/§17 상세·§8/§10 태그·
   §15 카테고리·§16 엑셀·§18 필드제약) 와 대조해 **[이미 함 / 갭 / 개선안]** 판정.

6. **개선 반영** —
   - **저위험·명백한 개선**(예시 문구·체크리스트·통과태그 추가 등)은 `/learn` 방식으로 LEARNED_RULES + memory 에 바로 반영.
   - **큰 변경**(템플릿 구조·가격산식·정책 해석)은 반영하지 말고 **Slack 에 "제안"으로만** 올려 사장님 확인받는다.
   - 데이터 진실성(§9) 유지 — 근거 불명확하면 반영 X.

7. **로그 기록** — `study_log.csv` 에 오늘 학습(date,topic,source_url,takeaway) 행 추가(중복 방지).

8. **Slack 다이제스트** — `python3 scripts/slack_notify.py --text "<요약>"` 로 `#new-item` 전송. 형식:
   ```
   📚 스마트스토어 공부 (YYYY-MM-DD) — <주제>
   • 오늘 배운 것: <핵심 2~3줄, 출처 링크>
   • 우리 현황: <이미 함 / 갭>
   • 개선안: <구체 액션 1~3개> (반영함 / 제안만)
   ```
   반영한 게 있으면 그 파일도 한 줄 명시.

## 실행 메모
- 헤드리스(cron)로도 도는 실행이라 **사용자 확인 없이 완주**한다(큰 변경만 Slack 제안으로 보류).
- 검색·읽기가 다 막히면 "오늘 소스 접근 실패" 1줄 Slack 보고 후 종료(로그엔 남김).
- 무거운 조사·정독은 서브에이전트(general-purpose/Explore)에 위임해도 됨. 반영·Slack·로그는 메인 소유.
- learn 반영 시 커밋/푸시(§learn 규칙) — 단 반영이 있을 때만.
