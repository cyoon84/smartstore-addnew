> 이 파일을 수정한 뒤, 내용을 복사해서 Cowork 앱 프로젝트 설정에 붙여넣어야 실제로 반영됩니다.

---

## 입력할 정보

제공할 항목:

- 사진
- 제품 원가
- HST 적용여부
- 고객이 주문할 때 내면 좋을 배송비
- 남기고 싶은 마진 (% 또는 $)

## 가격 계산 규칙

현재 시점 CAD → KRW 환율 적용하고,
제품 원가에 원하는 마진 더해서 네이버 스마트스토어 수수료 6.6% 적용해서 계산해줘.

## 제품명

- 50자 이내
- 상품명 끝에 **국가(캐나다/미국)는 넣지 않음** — 네이버가 원산지 필드 값을 `[원산지:캐나다]`로 자동 표시(중복). 국가는 원산지 필드로만.

## 모델명 (영어 원래 이름)

- 네이버 **모델명 필드**에 **영어 원래 제품명(공식 풀네임)**을 넣는다 — 영어/로마자 검색 매칭 + 랭킹 적합도 향상.
- **Cowork는 크롤 가능하므로 소스 페이지(Walmart·Amazon 등)에서 영어 원래 이름을 직접 가져온다.** (예: "Head & Shoulders Clinical Dry Scalp Rescue Shampoo")
- 그룹상품이면 옵션(변형)별로 모델명을 각각 가져온다.
- 크롤로도 확인 안 되면 추측해서 지어내지 말고 비워두거나 사용자에게 확인(데이터 진실성).

## 상세페이지 생성 규칙

### 파일 저장 위치 (데스크톱·모바일 dispatch 공통)

- 저장 폴더: `/Volumes/External/claude/smartstore-project/output/`
- 폴더 분리 X, 평탄 구조 유지 (Downloads 사용 안 함, 중첩 폴더 만들지 않음)
- 파일명 규칙 (슬러그 prefix):
  - `<ascii_slug>_detail.html`
  - `<ascii_slug>_product_info.json`
  - `<ascii_slug>_등록정보.md`
  - 참고 이미지: `<ascii_slug>_*.jpg` 등 같은 폴더에

### 허용/금지 태그

- ❌ `<div>`, `<style>`, `<script>` 사용 금지 (네이버 스마트스토어에서 제거됨)
- ✅ `<p>`, `<strong>`, `<br>` 등 기본 태그와 텍스트 위주로만 작성
- 이모지는 자유롭게 사용 가능

### 포함할 섹션

- 상품명 / 헤드라인
- 상품 소개 (간단한 설명)
- 이런 점이 좋아요 (핵심 포인트)
- 이런 분들께 추천해요
- 마무리 한 줄

### 제외할 섹션

- 상품 상세 정보 (스펙 테이블)
- 배송 안내
- Footer / 하단 안내문
